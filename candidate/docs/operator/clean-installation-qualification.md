# Qualification d’installation vierge

1. Construire le candidat depuis le commit gelé.
2. Déployer le package dans un répertoire vide.
3. Lancer la qualification sans données utilisateur.
4. Vérifier le statut `qualified`.
5. Contrôler le profil `production`, le mode démonstration désactivé et le commit source.
6. Vérifier l’intégrité de tous les fichiers.
7. Vérifier le démarrage `ready` et l’arrêt `stopped`.
8. Consigner le rapport et la décision opérateur.

Un statut `blocked` interdit la promotion du candidat. Aucune correction automatique du package n’est autorisée.
