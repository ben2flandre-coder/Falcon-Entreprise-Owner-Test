## Frontières de phase

EI-16.3 ne prétend pas livrer :

- le mécanisme propriétaire définitif de licence, prévu en EI-16.4 ;
- une synchronisation cloud ;
- une IA externe active ;
- les audits IA croisés, prévus en EI-16.5 ;
- une autorisation d'utiliser des données client réelles pendant le pilote documentaire.

## Règle de mise à jour

Tout changement d'un flux, libellé, rôle, format, procédure de récupération, profil de release ou limite connue doit mettre à jour :

1. le document concerné ;
2. `ei16-documentation-contract.json` si le chemin ou la preuve change ;
3. les tests fonctionnels correspondants ;
4. la baseline et les preuves de qualification dans l'issue de lot.

Une ligne ne peut être déclarée couverte que si son document principal et ses preuves existent sur le même SHA.
